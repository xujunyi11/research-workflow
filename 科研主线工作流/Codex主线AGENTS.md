# 科研实现约定

Codex 根据用户明确指定的最新版 `CODEX_TASK_vNNN.md` 修改基线仓库，不自行改变研究假设。

## 实现

- 先核对仓库、提交、配置、数据和评测入口；任务与代码冲突时停止猜测，输出带版本号的 `BLOCKER_vNNN.md`。
- 保留基线行为，以最小改动实现模块和配置开关；先复现基线，再实现新方案。
- 先做单卡冒烟测试，再按任务扩展，最多使用 4×RTX 4090。
- 执行与基线一致的训练、验证和测试逻辑；记录实际命令、配置、提交和随机种子。不得伪造未运行结果。

## 版本化交付

每次运行输出 `TRAINING_PACKET_vTTT_rRR.md`：`TTT` 对应任务版本，`RR` 为该任务下从 `01` 递增的运行号；永不覆盖旧数据包。代码、配置、打包脚本和阻塞文件也应可由提交或文件版本追溯。

在理解基线的训练、验证、测试流程后，自适应创建最简单可用的打包脚本和运行指令。数据包只需包含足以让 Work 判断实验的内容：任务/运行版本、代码提交、实际配置与命令、改动摘要、基线与新方案关键指标、必要的曲线或精选日志、失败信息、资源消耗和相关文件路径。

默认不打包数据集、权重、完整日志、缓存或密钥；只有诊断确实需要时才附加小型文件。失败运行同样生成数据包。Codex 只报告实现和观测事实，不代替 Work 作研究结论。
